# AI素养课程设计方案

> **时间：** 2025-12-24  
> **讨论者：** 张老师 & Sean  
> **类型：** 课程设计与问题解决  
> **状态：** 方案已确定 ✅  
> **项目：** 高中教师AI素养培训（6节课，分科目）

---

## 一、困境与约束条件

### 背景信息
- **身份：** 程序员转教师，刚接触AI教学
- **对象：** 高中9科老师（语数外物化生政史地）
- **课程：** 6节课，分科目进行
- **已完成：** 第1节（提示词）、第2节（AI教研员）

### 核心困境

#### 1. 教学者层面
- 刚接触AI，正在学习
- 不是学科老师，无法判断教学内容好坏
- 认同AI是协作伙伴，但要教把AI当工具的人

#### 2. 学员层面（老师）
- **能动性极差**：认为是学校安排，不是自己需要
- **计算机素养低**：不会装软件、配置环境、甚至不知道在浏览器输入网址
- **认知矛盾**：
  - 要求手把手教，但教了又不动手
  - 想要简单直接，但又问为什么
  - 觉得自己写得更好，AI没用

#### 3. 环境约束
- **不能科学上网**：无法用ChatGPT、Gemini、NotebookLM
- **国内AI不稳定**：
  - 生图抽卡游戏，现场可能翻车
  - 编程能力有限
  - 准备好的案例可能现场不一样
- **只能用**：豆包（最熟悉）、deepseek、kimi

#### 4. 课程要求
- 必须分科目（不能通用）
- 示例必须学科相关（否则借口没意义）
- 必须极简操作（baby step级别）
- 必须包含文生图和编程（老师刚需）

---

## 二、问题分析（矛盾论）

### 主要矛盾识别

**核心矛盾：**
> **需要设计课程 vs 不知道老师真实需求**

**次要矛盾：**
- 想教AI能力 vs 老师只想要配方
- 想讲原理方法 vs 老师不爱听概念
- 国内AI不稳定 vs 需要可靠演示

### 目标澄清

**最初设想（A）：** 理想状态 - 学会AI协作思维（不可能）  
**妥协状态（B）：** 记住6个配方，觉得有用 ✅ **选这个**  
**种子状态（C）：** 埋下种子自己探索（比B更难）  
**完成任务（D）：** 走过场（底线）

**确定目标：**
> 让老师记住几个可复制的配方，觉得这个有用，有课堂效果

## 三、解决方案

### 核心策略调整

#### 从"教能力"到"教配方"
- ❌ 不教：AI原理、提示词技巧、完整体系
- ✅ 要教：可复制操作步骤、具体场景应用

#### 从"判断好坏"到"现场验证"
- 教的是：操作方法（通用）
- 生成的是：学科内容（分科目）
- 判断标准：老师现场自己判断有没有用

#### 把"不稳定"变成"教学内容"
- 接受AI会失败
- 教会应对失败的方法
- 准备备选方案和救场材料

### 示例设计标准

一个合格示例必须满足：
```
✅ 条件1：极简操作（≤5步，每步都是"打开XX，输入XX"级别）
✅ 条件2：稳定可复现（豆包/deepseek能90%成功率）
✅ 条件3：即时验证（老师3秒内能判断有没有用）
✅ 条件4：学科相关（能套用到9个科目）
✅ 条件5：高频场景（老师每周/每月都会遇到）
```

## 四、编程课程方案（已验证可行）

### 方案1：化学分子结构可视化工具（主方案）✅

**验证状态：** 已在deepseek测试通过，非常顺滑

**完整步骤（6步迭代）：**

#### 步骤1：生成水分子（10分钟）
```
提示词："用HTML+SVG画一个水分子结构图，氧原子红色，氢原子白色，显示原子连接，代码写在一个文件里"

操作：
1. 打开deepseek
2. 输入提示词
3. 如果停止，点"继续生成"
4. 复制完整代码（确认最后是</html>）
5. 保存为 water.html
6. 双击打开查看
```

#### 步骤2：美化效果（8分钟）
```
提示词："给原子加上渐变色和阴影效果，看起来更立体"

操作：
1. 在同一对话继续输入
2. 复制新代码
3. 替换原文件
4. 刷新查看变化
```

#### 步骤3：添加交互功能（10分钟）
```
提示词："加一个输入框，我输入分子名称（比如CO2、CH4、NH3），就能显示对应的分子结构"

操作：
1. 继续对话
2. 复制代码
3. 保存
4. 测试输入不同分子
```

#### 步骤4：添加元素标签（7分钟）
```
提示词："在每个原子旁边显示元素符号（H、O、C等）和元素名称"
```

#### 步骤5：添加说明文字（5分钟）
```
提示词："在页面上方添加使用说明，告诉用户可以输入哪些常见分子"
```

#### 步骤6：优化样式（5分钟）
```
提示词："美化整体页面，添加标题、边框，让它看起来像一个专业工具"
```

## 五、课程话术与应对

### 关于"继续生成"的处理

**话术1（第一次出现）：**
> "大家看，AI生成的代码太长了，它自动停下来了。这很正常，咱们点一下这个'继续生成'按钮，让它把剩下的代码写完。"

**话术2（提前预告）：**
> "等会儿如果看到AI停下来了，下面有个'继续生成'按钮，点一下就行，它会接着往下写。"

**话术3（教学化）：**
> "这是AI的一个特点，代码太长时会分段生成。就像咱们写作文，写太长了要分段，对不对？AI也是一样。看到'继续生成'就点一下，很简单。"

### 失败应对话术

**处理失败的标准话术：**

❌ 不要说："AI很强大，什么都能做"  
✅ 应该说："AI有时候会出错，这很正常，咱们看看怎么应对"

❌ 不要说："按这个步骤就一定能成功"  
✅ 应该说："这是常用步骤，如果不行，咱们就这样调整"

**现场失败处理：**
```
第一次失败 → "你们看，它理解错了，咱们换个说法"
连续失败 → "豆包不太行，咱们试试deepseek"（现场切换）
还是失败 → "这个功能确实不太稳定，我这有个备用版本"（展示提前做好的）
```

### 课程流程模板（45分钟）

**1-5分钟：开场**
- 今天做什么（化学工具）
- 为什么有用（可以用在教学中）
- 怎么做（跟AI对话，分步骤）

**5-35分钟：主体演示**
- 方案1为主（分子结构）
- 每步5-6分钟
- 遇到问题现场解决

**35-40分钟：展示其他可能**
- 快速演示方案2/3/4的效果（提前做好的）
- "这些都是用同样的方法做的，你们课后可以试"

**40-45分钟：总结和答疑**
- 发放操作步骤文档
- 回答问题
- 鼓励尝试

### 灵活调整策略

| 情况 | 应对方案 |
|------|---------|
| **进度快（25分钟完成）** | 现场再做一个方案2或方案4 |
| **进度慢（40分钟才完成）** | 省略后几步，直接展示最终效果 |
| **主方案失败** | 立即切换到备选方案 |
| **老师不动手** | 改为演示+发文档，课后自己试 |

## 七、核心洞察与原则

### 设计原则

**示例标准（5个必须）：**
1. 极简操作（≤5步）
2. 稳定可复现（90%成功率）
3. 即时验证（3秒判断）
4. 学科相关（可套用9科）
5. 高频场景（常用）

**策略原则：**
- 不是判断好坏，而是让老师现场验证
- 不是追求完美，而是接受失败并应对
- 不是教原理，而是教可复制的配方
- 不是通用内容，而是学科相关示例

### 关键认知转变

**从"能力导向"到"问题导向"：**
- ❌ 文生文、文生图、编程、智能体（AI能做什么）
- ✅ 备课、出题、找图、写报告（AI能帮我做什么）

**从"系统学习"到"配方复制"：**
- ❌ 理解提示词原理、学会AI协作思维
- ✅ 记住操作步骤、照着做能出结果

**从"稳定演示"到"应对失败"：**
- ❌ 追求完美演示，失败就翻车
- ✅ 把失败变教学内容，教会应对
