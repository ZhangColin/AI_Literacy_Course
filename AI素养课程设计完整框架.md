## 一、核心理念

### 设计哲学

**务实理想主义**：在极端约束下找到可行解

**核心信念**：
- 接受现实约束（老师能动性差、AI不稳定、环境限制）
- 不追求完美课程，追求可行方案
- 让老师记住可复制的配方，觉得有用
- 在约束中做到极致

### 主要矛盾

**核心矛盾**：老师不知道写什么提示词 vs 需要用AI
- **解决方案**：第1课教方法（三要素），第2-5课给配方（直接能用的提示词）
- **矛盾转化**：从"不会写" → "会套用" → （如果用得好）→ "自己改"

**次要矛盾**：
- AI不稳定 vs 需要可靠演示 → 临场应变，准备足够多示例
- 不知道真实需求 vs 需要设计课程 → 第6课机动，现场验证后调整
- 每科不同 vs 需要标准化 → 框架统一，只换示例

## 二、标准6课框架（通用模板）

### 课程总览

| 课次 | 主题 | 核心目标 | 通用性 |
|------|------|---------|--------|
| 第1课 | 提示词方法论 | 教会三要素思考框架 | 100%通用，只换案例 |
| 第2课 | 文生文应用 | 给该科目/行业的现成提示词 | 需定制示例 |
| 第3课 | 文生图/音视频 | 按需决定，需求不强可换内容 | 需判断需求 |
| 第4课 | AI编程 | 生成该科目专用工具 | 理科必讲，文科选讲 |
| 第5课 | 智能体（Coze） | 搭建该科目专用助手 | 需定制场景 |
| 第6课 | 机动课 | 补前面内容+答疑+新需求 | 灵活机动 |

### 课程设计原则

**原则1：第1课是唯一有理论的课**
- 讲三要素方法论
- 用弱vs强提示词制造认知冲击
- 参照文旅版教师指南，只需替换案例
- 强调：三要素是思考框架，不是死格式

**原则2：第2-5课都是给配方**
- 准备足够多的示例提示词
- 提示词已经写好三要素，老师改几个关键词就能用
- 一句提示词能出当下最好效果
- 不讲原理，只讲操作

**原则3：示例数量灵活**
- 效率高的示例（5-10分钟）→ 可以多讲几个
- 效率低的示例（15-20分钟）→ 少讲几个
- 核心场景必讲，讲不完的放第6课

**原则4：每课都要有行动卡实战环节**
- 老师演示完 → 学员立即实战练习
- 准备行动卡（打印材料）：核心提示词模板+填空位
- 学员用真实场景练习
- 讲师巡场答疑
- 学员分享成果
- 上课效果实测：非常有效

**原则5：第6课是机动课**
- 前面讲不完的继续讲
- 收集到的老师问题集中答疑
- 现场发现的新需求临时演示

## 三、课程设计方法论

### 核心洞察

**老师不知道给什么样的提示词** - 这是所有问题的根源

**推论：**
- 我们教会基本方法（第1课：三要素）
- 我们给现成配方（第2-5课：直接能用的提示词）
- 我们灵活应对（第6课：处理遗留问题）

### 第一步：设计第1课（提示词方法论）

**目标**：让老师掌握三要素思考框架

**操作：完全套用文旅版，只需替换案例**

**需要准备：**
1. **弱vs强提示词案例**（该科目/行业的典型场景）
   - 弱提示词：真实但效果差
   - 强提示词：完整三要素，效果明显更好
   - 对比要有震撼效果（认知冲击）

2. **三要素讲解**（照搬文旅版）
   - 明确任务、丰富背景、细化要求
   - 用白皮书案例演示（可换成该科目案例）
   - 最后强调：三要素是思考框架，不是死格式

3. **行动卡实战**（照搬文旅版）
   - 学员用真实场景练习
   - 巡场答疑
   - 学员分享

4. **进阶技巧**（可选，照搬文旅版）
   - 风格仿写演示
   - 其他技巧简介

**重点：**
- ✅ 参照《提示词课程-教师教学指南》（文旅版）
- ✅ 只需替换弱vs强案例
- ✅ 其他话术、流程、时间分配都不变

**判断标准：**
- 弱提示词效果真的不好
- 强提示词效果明显更好
- 对比够震撼，老师觉得"原来如此"

---

### 第二步：识别并准备第2-5课的示例

**目标**：为第2-5课准备足够的示例提示词

**步骤1：识别典型场景（5-8个）**

**方法：**
- 列出该科目/行业的高频工作场景
- 判断AI能否解决（用8原则检验）
- 判断老师手头是否已有资源（有的话就不做）

**判断标准（新8原则）**：

**原5原则**：
1. ✅ 极简操作（≤5步，每步baby step级别）
2. ✅ 稳定可复现（国内模型90%成功率）
3. ✅ 即时验证（老师3秒内能判断有没有用）
4. ✅ 学科相关（能套用到具体学科）
5. ✅ 高频场景（老师常用）

**新3原则**（关键）：
6. ✅ **认知冲击**：让老师看到"原来可以这样"（不是平时也会问的）
7. ✅ **提示词价值体现**：展示提示词优化过程（不是一次性生成）
8. ✅ **避开已有资源**：不碰老师手头已有的（如教案模板、实验步骤模板）

**输出示例：**
```
【英语科第2课：文生文】场景清单

场景1：作文批改 ✅✅✅
- 高频场景：老师每周都要批改
- 刚需程度：高（费时费力）
- 讲解效率：快（10分钟）
→ 必讲

场景2：阅读理解出题 ✅✅
- 高频场景：出卷子时用
- 刚需程度：中（有资源但不够）
- 讲解效率：中（15分钟）
→ 必讲

场景3：单词例句生成 ✅
- 高频场景：备课时用
- 刚需程度：中
- 讲解效率：快（8分钟）
→ 时间够就讲

场景4：听力材料生成 ✅
- 高频场景：不太高频
- 刚需程度：中
- 讲解效率：中（12分钟）
→ 备选

...（准备5-8个场景）
```

---

**步骤2：为每个场景写提示词**

**关键原则：提示词形式灵活，不固定格式**

**形式1：完整三要素**（适合复杂任务）
```
【任务】...
【背景】...
【要求】...
```
**适用场景**：老师第一次用，不确定怎么问

**形式2：简化版**（适合简单高频任务）
```
批改这篇作文，学生是高二中等水平，
给出语法错误、词汇建议和分数预估：
...
```
**适用场景**：老师已经用熟了

**形式3：苏格拉底式提问**（适合探索式任务）
```
我想设计一个关于环保的写作课。
我的想法是...但我不确定...
你能帮我一步步分析吗？
```
**适用场景**：老师自己也不确定要什么

**形式4：多轮对话**（适合迭代优化）
```
第1轮："帮我批改这篇作文"
第2轮："第3个错误能详细解释吗？"
第3轮："给我5个替换词"
```
**适用场景**：不期待一次完美

**核心要求：**
- 提示词已经写好，老师改几个关键词就能用
- 一句提示词能出当下最好效果
- 什么形式合适就用什么形式

---

**步骤3：标注讲解效率**

为每个场景标注：
- **快**（5-10分钟）：一个提示词搞定，演示清晰
- **中**（10-15分钟）：需要2-3轮演示
- **慢**（15-20分钟）：需要多轮迭代或AI可能不稳定

**目的**：现场根据时间选择讲哪几个

---

**步骤4：准备材料清单**

每个场景准备：
1. **提示词**（什么形式合适就用什么）
2. **演示流程**（分几步，每步说什么）
3. **话术**（开场、过渡、应对失败）
4. **备选方案**（如果AI失败怎么办）
5. **行动卡**（学员实战用）
   - 核心提示词模板（留填空位）
   - 操作步骤（1-2-3）
   - 示例参考（学员不会时可参照）

---

### 第三步：技术验证

**验证标准**：
- 能做出来 = 通过（不追求完美）
- 做不出来 = 不用（换示例）

**判断示例质量的3个核心问题**（不需要懂学科）：

**问题1：核心教学点是否体现？**
- 操作后能看到/理解教学概念吗？

**问题2：操作是否有认知价值？**
- 学生操作后理解了什么？

**问题3：最简版本是否已足够？**
- 去掉炫酷效果，教学点还在吗？

**重要认知**：
- 简单 ≠ 不好
- 复杂 ≠ 好
- 教学价值才是标准
- 被效果丰富"忽悠"是陷阱

### 第四步：现场执行策略

**原则1：灵活调整讲解数量**
- 不固定"必须讲X个示例"
- 根据讲解效率和时间选择
- 核心场景必讲，其他看时间

**示例：第2课（90分钟）执行策略**
```
前30-40分钟：教学示例演示
- 讲解2-3个核心场景（必讲）
  场景1：10分钟（效率快）
  场景2：15分钟（效率中）
  场景3：10分钟（备选，看时间）
- 演示：讲师用提示词现场演示
- 强调关键点和注意事项

判断时间：
- 如果讲得快 → 加1-2个备选场景
- 如果讲得慢 → 核心场景讲完就收

中间40分钟：行动卡实战
- 发放打印材料（行动卡）
- 学员用真实场景练习（用提供的提示词模板）
- 讲师巡场答疑（重点）
- 学员可以互相交流

最后10-20分钟：
- 学员分享成果（2-3个）
- 集中答疑
- 总结要点
```

**原则2：应对AI不稳定**
- 步骤提前/跳过 → 顺着走，现场调整话术
- 完全失败 → 切换备选方案或备用示例
- 关键心态：**把失败变教学内容**
- 备用话术："大家看，AI在猜，但我们明确告诉它会更准"

**原则3：收集反馈迭代**
- 老师觉得有用 → 记录，下次多用
- 老师觉得没用 → 记录，换掉
- 现场发现的新需求 → 记录，第6课讲

---

### 第五步：第6课机动设计

**第6课的作用：**
1. 前面讲不完的场景继续讲
2. 前面收集的老师问题集中答疑
3. 现场发现的新需求临时演示

**准备内容：**
- 前面没讲的场景示例（已准备好）
- 高频问题库（10个标准答案）
- 备用示例库（5个不同方向，根据需求选）
- 万能话术（如果实在不知道怎么答）

**执行策略：**
- 提前询问老师："前面哪些内容想再深入？有什么问题？"
- 根据反馈灵活安排
- 时间够就多演示，时间不够就答疑

---

### 标准课程时间结构（90分钟）

**适用于：第2-5课（第1课参照文旅版教师指南）**

```
【30-40分钟】教学示例演示
├─ 开场（2分钟）
├─ 场景1演示（10分钟）
├─ 场景2演示（15分钟）
└─ 场景3演示（10分钟，备选）

【40分钟】行动卡实战（核心环节）
├─ 发放打印材料（2分钟）
├─ 学员实战练习（30分钟）
│  └─ 讲师巡场答疑（重点）
└─ 学员交流讨论（8分钟）

【10-20分钟】分享与答疑
├─ 学员成果分享（10分钟，2-3个）
├─ 集中答疑（5分钟）
└─ 总结要点（5分钟）
```

**关键原则：**
- 演示不要超过40分钟（否则学员没时间实战）
- 实战时间至少30分钟（学员需要充足时间练习）
- 巡场答疑是重点（一对一解决问题）
- 学员分享激励效果好（树立榜样）

## 三、附录/扩展机制

### 机制说明

**什么是附录/扩展**：
- 不在主课讲，但准备好的示例
- 老师问了可以演示
- 或在提示词课"带一嘴"

**为什么需要**：
1. 主课聚焦核心（必讲内容）
2. 附录覆盖边缘（可能需要的场景）
3. 灵活应对（老师问了不慌）
4. 环境不受限（能用什么用什么，做得出来就演示）

**准备内容**：
- 每个学科识别1-2个附录示例
- 记录操作步骤
- 如果做不出来，直接说"目前做不到"也是答案

## 四、已验证的成功案例

### 化学科 × AI编程

**方案**：化学分子结构可视化工具

**验证状态**：✅ 已验证成功

**步骤设计**（6步迭代，实际走4步）：
1. 生成水分子（10分钟）
2. 美化效果（8分钟）
3. 添加交互功能（10分钟）
4. 添加元素标签（7分钟）
5. 添加说明文字（5分钟）- 未执行
6. 优化样式（5分钟）- 未执行

**关键发现**：
- deepseek稳定性好
- 6步太多，4步刚好（因为老师计算机素养差+时间紧）
- "继续生成"需要提前告知
- 备选方案很重要（虽然没用上）

**老师反应**：
- 整体效果达到预期
- 个别极端老师不考虑
- 有互动（"这个怎么没出来"）
- 示例产生的问题属于现场可控

### 生物科 × AI编程

**方案**：孟德尔遗传计算器

**验证状态**：✅ 已验证成功

**为什么成功**：
- 纯计算，不涉及复杂图形
- 逻辑清晰（孟德尔遗传）
- 类似化学浓度计算器
- 老师计算机素养相对高

**关键发现**：
- 计算类示例稳定性高
- 可复制到其他理科（物理公式、数学函数）

### 核心结论

**理科编程路径可复制**：
- 化学成功 + 生物成功 = 理科编程可行
- 计算类工具成功率最高
- 可视化工具相对不稳定但可用

**影响因素**：
- 老师计算机素养影响很大
- 需要准备不同难度的示例

## 四、具体实施指南

### 如何判断第3课讲什么

**第3课：文生图/音视频（需判断需求）**

**判断标准：**
| 判断项 | 如果需求强 | 如果需求弱 |
|-------|----------|----------|
| 该科目是否高频用到图片/音视频 | 是 → 讲 | 否 → 不讲 |
| 老师手头是否已有足够资源 | 没有 → 讲 | 有 → 不讲 |
| AI生成效果是否满足需求 | 能满足 → 讲 | 不满足 → 不讲 |

**如果需求弱，第3课改成什么？**
- 方案A：更多文生文示例（把第2课讲不完的继续讲）
- 方案B：文生文进阶技巧（风格仿写、多轮优化等）
- 方案C：提前讲第4课或第5课

**示例判断：**
- 英语科：听力材料生成需求高 → 讲音频生成
- 数学科：图片/音视频需求低 → 改成更多文生文或提前讲编程
- 语文科：古诗配图需求中等 → 可讲可不讲，现场判断

---

### 如何判断第4课讲什么

**第4课：AI编程（需判断适用性）**

**判断标准：**
- 理科（数理化生）：✅✅✅ 必讲（生成学科工具）
- 文科（语史地政）：⚠️ 选讲（需要找到合适场景）
- 英语：⚠️ 选讲（单词测试小工具？）

**如果不适合讲编程，改成什么？**
- 方案A：更多该科目的AI应用示例
- 方案B：智能体（提前到第4课）
- 方案C：综合应用（把前面的串起来）

---

### 第5课：智能体（Coze）

**核心策略：**
- 搭建该科目/行业专用的AI助手
- 演示Coze的基本搭建流程
- 重点：预设提示词（回到提示词的重要性）

**准备内容：**
- 2-3个该科目的智能体示例
- 搭建流程演示（10-15分钟）
- 提示词设计演示（关键环节）

---

### 第6课：机动课设计

**三大功能：**
1. **补充前面没讲完的**
   - 第2-5课准备了但没讲的示例
   - 老师感兴趣但时间不够的内容

2. **集中答疑**
   - 前5课收集的问题
   - 准备10个高频问题的标准答案

3. **新需求快速响应**
   - 现场发现的新场景
   - 准备5个不同方向的备用示例

**执行策略：**
- 课前调研：询问老师想深入哪些内容
- 灵活安排：根据需求现场调整
- 时间分配：50%补充+30%答疑+20%新需求

## 五、工作流程总结

### 设计新科目/行业课程的完整流程

**第1步：设计第1课（1-2小时）**
```
1. 完全套用文旅版教师指南
2. 只需设计1个弱vs强提示词案例
   - 选该科目/行业的典型场景
   - 弱提示词：真实但效果差
   - 强提示词：完整三要素，效果明显更好
3. 测试对比效果（确保认知冲击明显）
4. 其他内容（三要素讲解、行动卡、进阶技巧）全部不变
```

**第2步：识别第2-5课的场景（2-3小时）**
```
1. 列出该科目/行业的高频工作场景（10-15个）
2. 用8原则筛选（刚需、高频、老师手头没资源）
3. 判断讲解效率（快/中/慢）
4. 选出5-8个典型场景
5. 分配到第2-5课
   - 第2课：文生文（3-5个场景）
   - 第3课：文生图/音视频（2-3个，或不讲）
   - 第4课：AI编程（2-3个，或不讲）
   - 第5课：智能体（2-3个）
```

**第3步：为每个场景写提示词（3-5小时）**
```
1. 为每个场景设计提示词
   - 什么形式合适就用什么（完整三要素/简化版/苏格拉底式/多轮对话）
   - 提示词写好，老师改几个关键词就能用
2. 标注讲解效率（快/中/慢）
3. 准备演示流程和话术
4. 准备备选方案（AI失败怎么办）
```

**第4步：技术验证（2-3小时）**
```
1. 用deepseek/豆包测试每个提示词
2. 确保90%成功率
3. 做不出来的 → 换示例或准备备用方案
4. 效果好的 → 记录，作为核心示例
```

**第5步：准备材料（2-3小时）**
```
1. 教师指南（第1课照搬文旅版，第2-5课写简要流程）
2. 学员打印材料（每课必备）
   - 核心提示词模板（带填空位）
   - 行动卡（实战练习用）
   - 操作步骤（1-2-3）
3. 在线手册（扩展示例+常见问题）
4. 备用材料（救场用的完整文件/截图）
```

**总计时间：10-16小时（设计一个完整的6课体系）**

### 现场执行策略

**课前准备（每次上课前30分钟）**
```
1. 测试AI工具（deepseek/豆包可用吗）
2. 测试网络和投屏
3. 准备好演示账号
4. 打开所有准备的提示词文档
```

**第1课：按文旅版教师指南执行**
```
- 认知冲击（5分钟）
- 三要素讲解（20分钟）
- 行动卡实战（50分钟）
- 进阶技巧（15分钟）
```

**第2-5课：灵活执行**
```
1. 教学示例演示（前30-40分钟）
   - 讲核心场景（标注"必讲"的）
   - 看进度：快→加备选场景，慢→核心场景讲完就收
   
2. 行动卡实战（中间40分钟）
   - 发放打印材料
   - 学员用真实场景练习
   - 讲师巡场答疑（重点环节）
   
3. 学员分享+答疑（最后10-20分钟）
   - 2-3个学员分享成果
   - 集中答疑
   
4. 讲不完的 → 记录，第6课讲
```

**第6课：机动执行**
```
1. 课前询问老师需求
2. 根据反馈灵活安排：
   - 补充前面内容（50%）
   - 集中答疑（30%）
   - 新需求演示（20%）
```

---

### 迭代优化流程

**每次上课后：**
```
1. 记录哪些示例效果好（下次多用）
2. 记录哪些示例效果差（换掉）
3. 记录老师问的高频问题（加入题库）
4. 记录AI失败的场景（改进或换示例）
5. 更新材料和话术
```

**积累成果：**
```
- 每个科目/行业的成功示例库
- 可复用的提示词模板
- 应对AI不稳定的话术库
- 高频问题的标准答案库
```

## 七、关键认知和原则

### 核心认知

**1. 关于目标**
- 不是教AI能力，是教配方
- 不是让所有老师都用，是让想用的老师会用
- 不是追求完美，是在约束中可行

**2. 关于提示词**
- 老师不知道写什么提示词 - 这是根本问题
- 三要素是思考框架，不是死格式
- 提示词形式灵活，什么合适用什么
- 第1课教方法，第2-5课给配方

**3. 关于示例数量**
- 不固定数量，看讲解效率
- 效率高的多讲，效率低的少讲
- 核心场景必讲，讲不完的放第6课
- 关键是"足够解决典型问题"，不是"面面俱到"

**4. 关于行动卡（每课必备）**
- 教学示例演示 + 行动卡实战 = 完整课程结构
- 讲师演示完，学员立即实战练习
- 行动卡提供提示词模板（留填空位），学员用真实场景填
- 巡场答疑是关键环节
- 上课效果实测：非常有效

**5. 关于AI不稳定**
- 这是客观事实，短期无解
- 应对方式：临场应变 + 备选方案
- 把失败变教学内容

**6. 关于示例判断**
- 没有学科知识时，用3问题框架判断
- 技术实现：能做出来 = 通过
- 教学价值：老师评判 = 终极标准

**7. 关于效果评估**
- 简单 ≠ 不好，复杂 ≠ 好
- 教学价值才是标准
- 不要被视觉效果忽悠

### 行为原则

**1. 奥卡姆剃刀**
- 删掉解决不了的问题（老师不用、绝对公平）
- 删掉不必要的复杂度
- 保留最小可用方案

**2. 务实理想主义**
- 承认约束是真实的
- 在约束中找空间
- 做到当下条件的极致

**3. 矛盾论思维**
- 识别主要矛盾和次要矛盾
- 聚焦主要矛盾
- 预判矛盾转化

**4. 迭代优化**
- 接受不完美
- 快速验证
- 持续改进

## 十、附录

### 成功话术案例

#### 开场话术（强调可迁移）

**化学 → 生物**：
> "各位生物老师，今天我们继续学AI编程，但做的是生物工具。
> 
> 昨天化学老师做了分子结构可视化，今天我们做遗传计算器。
> 
> 你们会发现：**方法是完全一样的**——都是和AI对话，一步步迭代。但**内容完全不同**。
> 
> 这就是AI的特点：**学会方法，可以应用到任何学科**。"

#### 失败应对话术

**第一次失败**：
> "你们看，AI理解错了，咱们换个说法"

**连续失败**：
> "豆包不太行，咱们试试deepseek"（现场切换）

**还是失败**：
> "这个功能确实不太稳定，我这有个备用版本"（展示提前做好的）

### 工具和环境

**推荐AI工具**：
- deepseek（最稳定）
- 豆包（最熟悉）
- kimi（备用）

**不能用**：
- ChatGPT、Gemini、NotebookLM（无法科学上网）

**开发环境**：
- HTML单文件（最简单）
- 本地浏览器打开（不需要服务器）
