# PromptX角色与工具系统总览

## 什么是PromptX？

PromptX是一个基于MCP（Model Context Protocol）的AI角色与工具管理系统，让AI能够通过自然语言对话完成复杂任务。

**核心理念：**
- **Chat is All You Need** - 对话即一切，无需复杂界面
- **AI是用户** - AI使用工具，人类指挥AI
- **单一真相源** - 每个概念只定义一次
- **模块化编排** - 清晰的层次和职责分离

**创始人：**Sean（deepractice.ai创始人&CEO）

## 系统架构

```
人类（决策者）
  ↓ 自然语言指令
AI角色（执行者）
  ↓ 工具调用
ToolX运行时
  ↓ 执行操作
外部系统/文件
```

### 三层资源体系

**系统级（System）：**
- PromptX内置资源
- 只读，不可修改
- 通用角色和工具
- 位置：PromptX安装目录

**项目级（Project）：**
- 项目特有资源
- 需要project工具激活
- 团队共享
- 位置：{project}/.promptx/resource/

**用户级（User）：**
- 用户自定义资源
- 个人专属
- 最高优先级
- 位置：~/.promptx/resource/

## 核心组件

### 1. DPML协议

**PromptX专有的角色定义标准**

**核心标签：**
```xml
<role>         <!-- 角色根容器 -->
<personality>  <!-- 思维模式容器 -->
<principle>    <!-- 行为准则容器 -->
<knowledge>    <!-- 私有知识容器 -->
```

**协议标签：**
```xml
<thought>      <!-- 思维协议文件 -->
<execution>    <!-- 执行协议文件 -->
<knowledge>    <!-- 知识协议文件 -->
```

**三层架构：**
- **Thought（思维模式）**：如何思考
- **Execution（执行流程）**：如何做事
- **Knowledge（知识体系）**：私有信息

### 2. ISSUE范式

**Sean原创的智能协作方法论**

| 步骤 | 含义 | 目标 | 时长 |
|------|------|------|------|
| **I**nitiate | 发起 | 接收需求，识别认知模式 | 10秒 |
| **S**tructure | 结构化 | 内部决策，选择框架 | 10秒 |
| **S**ocratic | 苏格拉底式对话 | 友好探索，收集信息 | 40秒 |
| **U**nify | 统一 | 整合方案，形成设计 | 30秒 |
| **E**xecute | 执行 | 生成交付，验证可用 | 30秒 |

**总时长：**2分钟内完成

**应用场景：**
- 女娲创建角色
- Writer撰写内容
- 需求分析
- 产品设计

### 3. Monogent理论

**Sean提出的单智能体认知系统**

**核心观点：**
- 不需要多智能体
- 单一AI通过角色和记忆实现专业化
- Recall-Answer-Remember认知循环

**认知循环：**
```
接收任务
  ↓
Recall（检索记忆）
  ↓
Answer（基于记忆+预训练回答）
  ↓
Remember（保存新知识）
  ↓
完成任务
```

**记忆网络：**
- Engram（记忆痕迹）：最小记忆单元
- Schema（概念序列）：关键词网络
- Strength（权重）：重要性和激活频率
- DMN（默认模式网络）：全景扫描

## 快速开始

### 1. 查看可用资源

```
使用promptx_discover工具：

discover({ focus: 'all' })      // 查看所有资源
discover({ focus: 'roles' })    // 只查看角色
discover({ focus: 'tools' })    // 只查看工具
```

### 2. 激活角色

```
使用promptx_action工具：

action({ role: 'luban' })       // 激活鲁班
action({ role: 'nuwa' })        // 激活女娲
action({ role: 'sean' })        // 激活Sean
```

### 3. 使用记忆系统

```
# DMN全景扫描
recall({ role: 'luban', query: null })

# 检索特定记忆
recall({ role: 'luban', query: 'ToolX 开发' })

# 保存新记忆
remember({
  role: 'luban',
  engrams: [{
    content: '今天学到的关键知识',
    schema: '关键词1 关键词2 关键词3',
    strength: 0.7,
    type: 'ATOMIC'
  }]
})
```

### 4. 使用工具

```
通过promptx_toolx工具调用：

# 查看工具手册
toolx({ yaml: 'tool: tool://filesystem\nmode: manual' })

# 执行工具
toolx({ yaml: 'tool: tool://filesystem\nmode: execute\nparameters:\n  action: read\n  path: /path/to/file.txt' })
```

### 5. 绑定项目

```
# 激活项目级资源
project({ workingDirectory: '/path/to/project' })
```

## 设计哲学

### 1. Chat is All You Need

**核心观点：** 自然语言对话是最好的人机交互方式

**实践：**
- 无需复杂配置界面
- 说话就能完成任务
- AI理解意图并执行
- 渐进式探索需求

### 2. AI是用户，人类是指挥官

**层级关系：**
```
人类（决策者）
  ↓ 指令
AI（执行者）
  ↓ 工具调用
软件/系统（被操作对象）
```

**实践：**
- 工具不是给人用的，是给AI用的
- 人通过对话指挥AI
- AI通过工具操作系统
- 降低人类学习成本

### 3. 单一真相源

**原则：** 每个概念只在一处定义

**实践：**
- DPML的引用机制
- 5行以上独立成文件
- 引用而非重复
- 修改一处全局生效

### 4. 奥卡姆剃刀

**原则：** 如无必要，勿增实体

**实践：**
- 删除所有非必要内容
- 每个词都要审视
- 能简短不冗长
- Token效率最大化

### 5. 2分钟原则

**目标：** 快速响应，立即可用

**实践：**
- 女娲2分钟创建角色
- 鲁班3-5分钟开发工具
- ISSUE范式2分钟完成探索
- 快速迭代优化
